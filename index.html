How it works

    Pick an engine

    OpenAI API for the cloud voices (needs your key), or Local (offline) to run Piper VITS right in the browser — no network after the first download.

  2. 02

    Drop text or files

    Type directly or upload .txt files. Each file keeps its own audio output; when you have two or more, a merged track is built on top automatically.

  3. 03

    Convert & download

    Live progress, per-item audio players, and one-click downloads. Cost estimate updates as you edit.

FAQ

Is my OpenAI API key sent anywhere? +

The key is stored in your browser's localStorage and sent only with each TTS request, which is routed through a thin /api/tts Vercel Edge proxy to OpenAI. Nothing is logged on the server side.

Does the offline engine really work without internet? +

Yes — once the Piper model is downloaded (~63 MB per voice, cached in the browser's Origin Private File System), synthesis runs fully on your device via onnxruntime-web and a WebAssembly build of piper_phonemize. No TTS request leaves the tab.

Which OpenAI models and voices are supported? +

tts-1, tts-1-hd, and gpt-4o-mini-tts. The voice list auto-filters to the voices that model actually supports — so picking tts-1 shows the classic six, and gpt-4o-mini-tts unlocks ash, ballad, coral, sage, verse on top.

Which languages does the offline engine support? +

Polish and English (US) out of the box, via Piper voices from rhasspy's canonical catalog. More languages could be added by registering their voiceId in src/types/index.ts.

How accurate is the cost estimate? +

For tts-1 / tts-1-hd it's exact (OpenAI bills per character). For gpt-4o-mini-tts it's an estimate — tokens per character are approximated (~4 chars per input token, ~1.8 audio tokens per character). Actual billing is whatever OpenAI charges.

What browsers are supported? +

Any modern evergreen browser for the OpenAI path. The offline engine additionally needs OPFS (Chrome, Edge, Safari 16.4+, Firefox 111+) and enough memory to run onnxruntime-web.
